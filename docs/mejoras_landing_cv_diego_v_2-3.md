# Especificación Técnica Mejorada — Landing Page CV Diego De Asís Ruiz

## Objetivo del Documento

Este documento redefine y mejora la especificación técnica original de la landing page personal de Diego De Asís Ruiz, incorporando mejoras estratégicas, visuales y de conversión orientadas a construir una experiencia digital premium alineada con perfiles senior de producto y tecnología.

La intención no es crear únicamente un portfolio visualmente atractivo.

El objetivo es transmitir:

- Seniority.
- Capacidad de liderazgo.
- Experiencia enterprise.
- Sensibilidad de producto.
- Perfil internacional.
- Capacidad técnica + visión de negocio.

---

# 5.1 Incorporar KPIs y Escala Real

## Problema detectado

La narrativa mejora mucho con el enfoque outcome-driven, pero sigue faltando percepción de escala.

Para perfiles Product Owner senior, los números generan credibilidad inmediata.

---

# Recomendación

Cada proyecto importante debería incluir al menos:

- usuarios impactados,
- mercados,
- volumen operativo,
- tamaño de equipos,
- releases,
- adopción,
- eficiencia,
- tiempo reducido,
- stakeholders.

---

## Ejemplos recomendados

```text
Unificación de procesos CRM para +500 usuarios comerciales en 5 mercados internacionales.
```

```text
Coordinación de roadmap Salesforce entre equipos distribuidos en Europa y América.
```

```text
Implementación de trazabilidad internacional de pagos para una de las mayores entidades financieras de España.
```

---

## Regla recomendada

Cada bloque “Selected Work” debería responder:

```text
¿Qué escala tenía?
¿Qué cambió?
¿Qué impacto generó?
```

# 5.6 Dirección de Motion y Animaciones

## Recomendación crítica

La experiencia debe sentirse:

```text
silenciosa,
fluida,
editorial,
precisa.
```

NO:

```text
llamativa,
startup,
interactiva en exceso.
```

---

# Animaciones recomendadas

## Priorizar

- staggered fade-ins,
- opacity transitions,
- micro-movements,
- soft parallax,
- reveal animations.

---

## Evitar

- rebotes,
- rotaciones,
- elastic animations,
- overscroll exagerado,
- efectos 3D,
- motion agresivo.

---

# Timing recomendado

```css
transition-duration: 400ms–700ms;
transition-timing-function: cubic-bezier(0.16, 1, 0.3, 1);
```

---

# Filosofía correcta

El usuario no debería “notar” las animaciones.

Simplemente debería sentir:

```text
refinamiento.
```

# 10. Stack Técnico Recomendado

## Stack final recomendado

```text
Astro + TailwindCSS + Motion One
```

---

# Justificación

## Astro

- performance excelente,
- SEO muy sólido,
- HTML estático,
- escalabilidad futura,
- posibilidad de añadir blog/proyectos.

---

## TailwindCSS

- consistencia visual,
- velocidad de desarrollo,
- responsive simplificado,
- mantenimiento sencillo.

---

## Motion One

- muy ligero,
- ideal para microinteracciones,
- elegante,
- menos pesado que GSAP.

# 11. Mejoras Frontend Modernas

## Scroll-Driven Animations

Usar:

```css
animation-timeline: view();
```

cuando sea posible.

Con fallback progresivo.

---

## Variable Fonts

Beneficios:

- menor peso,
- mejor rendering,
- refinamiento tipográfico.

---

## Container Queries

Más limpias y escalables.

```css
@container (min-width: 600px)
```

---

## View Transitions API

Ideal para:

- navegación suave,
- cambios de estado,
- microtransiciones premium.
